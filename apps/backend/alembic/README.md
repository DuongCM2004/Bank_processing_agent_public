# Alembic Migrations

This directory contains the PostgreSQL schema migration setup for the Ops Agent backend.

Structure:

- `env.py`: Alembic runtime integration with `ops_agent.config.get_settings()` and SQLAlchemy metadata.
- `script.py.mako`: revision template used for new migration files.
- `versions/`: ordered migration revisions. Filenames follow `revision_slug.py`.

Conventions:

- Alembic always resolves the database URL from `OPS_AGENT_POSTGRES_DSN` via backend settings.
- Revisions are stored under `alembic/versions`.
- Revision filenames use the revision id plus a short slug, for example `20260415_0001_initial_backend_schema.py`.
- Empty autogenerated revisions are suppressed in `env.py` so the history stays clean.

Local / dev usage:

1. Copy `.env.example` to `.env` inside `apps/backend`.
2. Ensure local PostgreSQL is available at the configured DSN.
3. Run migrations from `apps/backend`:

```powershell
py -m alembic upgrade head
```

4. Generate a new revision when models change:

```powershell
py -m alembic revision --autogenerate -m "describe change"
```

5. Downgrade one step if needed:

```powershell
py -m alembic downgrade -1
```

Initial migration:

- `20260415_0001_initial_backend_schema.py` creates the MVP schema for identity, cases, documents, extraction outputs, findings, decisions, manual review actions, and audit events.
